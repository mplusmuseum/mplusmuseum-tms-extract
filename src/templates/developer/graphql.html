Sigg Online

How to get started with Exhibitions and so on

If you are trying to work with Sigg Online data, you have a few more graphQL queries at your disposal. You also have concepts which are Artists and Artist Groups, and Exhibitions.

Concepts

To get all concepts you used a query such as

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.concepts ''}}}

And to fetch a single concept you can get it by id. Id 5 is that of the "Cultural Revolution Art" Artists movement

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.conceptNoObjects '(id: 5)'}}}

As usual you can get all objects for a concept two ways, you can ask the concept endpoint to incliude object data, and you can pass in the normal filters which will pass through to effect the objects, like this...

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.conceptWithObjects '(id: 5, per_page:2, page:0)'}}}

...or you can ask the objects endpoint to filter on the concept...

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.objectsMediumWithConcepts '(concept: 5, per_page:2, page:0)'}}}

Exhibitions

Exhibitions work the same way, you can get all exhibitions...

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.exhibitions ''}}}

Or you can just get the one we are interested in, Siggs Online is Exhibition 95 this is a minimal example...

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.exhibition '(id:95)'}}}

Again, you can do this via the objects endpoint. Here's a more fleshed out example for all all objects in the Sigg Online exhibition with the category of "Painting" shown 10 per page.

{{i18n.documentation.run-in-playground}} 

{{{graphQLQuery queries.objectsLargeWithExhibitions '(exhibition:95, per_page:10, page:0, category:"Painting")'}}}

Which should give you everything you need to string together showing objects for the exhibition. You will have to do some filtering to match the labels and bios with the exhibition using the "purpose" label.

What is missing

Currently the system is missing images (use https://placekitten.com/) for the moment. We are also missing the timeline data, both of these will be coming shortly.

The graphQL endpoints and this documentation will continue to evolve over the next few weeks, hang tight!
