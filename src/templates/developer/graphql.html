About GraphQL

GraphQL is a data query language you'll use to request information from the M+ data store. Traditionally GraphQL allows two types of operations, queries and mutations these can be thought of as read and write operations.

As well as one more operation called introspection , which allows you to ask the GraphQL endpoint about itself.

Below you will find examples of various calls to can make to the GraphQL endpoint.

Communicating with GraphQL

The single endpoint for all GraphQL calls is...

{{config.graphql.host}}/graphql

GraphQL operations consist of multiline JSON, an example "Hello World" GraphQL query is show below:

Run in the playground 

{{{graphQLQuery queries.hello ''}}}

To query GraphQL using cURL, make a POST request with a JSON payload as --data-binary. You must also include your API token as bearer authorization. 

{{{curlQuery queries.hello '' config.graphql.host user.apitoken}}}

Using Nodejs it may look like this, making sure to pass in your API token as a bearer token as shown. 

{{{nodeQuery queries.hello '' config.graphql.host user.apitoken}}}

GraphQL playground

Probably the easiest way to try out GraphQL queries is to use the playground which can be found here: {{config.graphql.host}}/playground

You can see an example query in the playground here, you will need to press the "run" button to see the results of the query.

Below you'll find a list of the current queries you can make against the data store.

Introspection

The introspection operation is the only operation you can call using GET. Introspection allows you to query the GraphQL schema for details about itself. Query __schema to list all types defined in the schema and get details about each:

Run in the playground 

{{{graphQLQuery queries.schema}}}

Query __type to get details about any type:

Run in the playground 

{{{graphQLQuery queries.type '(name: "Object")'}}}

Objects

This query selects all of the objects and can be filtered by certain fields shown below. Note: filters are additive rather than or, filtering two fields will only return records that match both.

Valid filters are: page, per_page, sort_field, sort, [ids], classification.area, classification.category, medium, displayDate, beginDate, endDate and lang

Valid values for the sort_field are id, objectNumber, sortNumber, title, displayDate, beginDate, endDate, classification.area and classification.category, values for sort are asc and desc.

The only valid values for lang are en, zh-hant and zh-hans. The default is en, if a record doesn't have values that match the selected language then it'll return the en version if there is one.

Here is the simplest call to get all object ids

Run in the playground 

{{{graphQLQuery queries.objectsMini ''}}}

cURL 

{{{curlQuery queries.objectsMini '' config.graphql.host user.apitoken}}}

NodeJS. 

{{{nodeQuery queries.objectsMini '' config.graphql.host user.apitoken}}}

Pagination and filter

Here we are asking for slightly more information, note how we can use the per_page filter to limit the number of results.

Run in the playground

cURL 

{{{curlQuery queries.objectsMedium '(page:3, per_page: 5, area: \\\"Moving Image\\\")' config.graphql.host user.apitoken}}}

NodeJS. 

{{{nodeQuery queries.objectsMedium '(page:3, per_page: 5, area: "Moving Image")' config.graphql.host user.apitoken}}}

Areas

You can fetch all the valid areas which you can then use to filter objects by. For example running the query below will return a handful of areas, which you would then use in an Object query with (area: "xxxxxxxx") where "xxxxxxxx" is one of the areas returned by this query.

Valid filters are per_page, lang, sort and sort_field.

Valid values for the sort_field are title and count. Valid values for the sort are asc and desc.

The only valid values for lang are en, zh-hant and zh-hans. The default is en, if a record doesn't have values that match the selected language then it'll return the en version if there is one.

The per_page filter will limit the number of results, for example (per_page: 10) will return just 10 records. Unlike Objects you cannot paginate through the Areas data set.

Run in the playground 

{{{graphQLQuery queries.areas '(lang: "zh-hant", sort_field: "count", sort: "asc")'}}}

cURL 

{{{curlQuery queries.areas '(lang: \\\"zh-hant\\\", sort_field: \\\"count\\\", sort: \\\"asc\\\")' config.graphql.host user.apitoken}}}

NodeJS. 

{{{nodeQuery queries.areas '(lang: "zh-hant", sort_field: "count", sort: "asc")' config.graphql.host user.apitoken}}}

Constituents

Constituents are people and institutions connected to an object, for example they may be the artists or the seller or buyer, they may be the curator or the institution the displays the object.

This query selects all of the constituents and can be filtered by certain fields shown below. Note: filters are additive rather than or, filtering two fields will only return records that match both.

Valid filters are: page, per_page, sort_field, sort, [ids], gender, beginDate, endDate, nationality and lang

Valid values for the sort_field are id, name, sortNumber, alphaSortName, gender, beginDate, endDate and nationality, values for sort are asc and desc.

The only valid values for lang are en, zh-hant and zh-hans. The default is en, if a record doesn't have values that match the selected language then it'll return the en version if there is one.

Nesting objects in constituents and constituents in objects

When you are searching for objects you can nest a limited version of constituents in the results. If you are requesting Objects you can get the constituents with the following query.

Run in the playground 

{{{graphQLQuery queries.objectsConstituent ''}}}

If you are asking for a single Object then you can get a bit more information about the constituents, see the example below.

Run in the playground 

{{{graphQLQuery queries.objectConstituent '(id: 119)'}}}

You cannot keep nesting objects and constituents beyond the first two levels, because they would just keep including each other forever.

You can also next constituents inside objects, but again with limited information returned and only down two levels.
